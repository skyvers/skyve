Free editorial template colour palettes (no paid theme required)
================================================================

The editorial template supports selectable colour palettes driven by the theme colour
on the UxUi in router.UxUis, e.g.

	UxUi.newPrimeFaces("phone", "editorial", "skyve", "emerald")

Available colours: "blue" (default), "indigo", "emerald", "custom" and "dark".

The light palettes follow the browser/operating system colour scheme preference and switch
to the dark palette in dark mode.
"dark" forces the dark palette and PrimeFaces arya theme permanently.
"custom" is a starter corporate palette (#004B88) intended to be copied and edited per project.

Dark logos: if a customer resource exists named <logoName>-dark.<extension> alongside the
configured logo (e.g. mylogo-dark.png for mylogo.png), it is served automatically whenever
the browser prefers dark mode or "dark" is selected permanently.

The exact PrimeFaces base theme and its colour palette are configured separately:
1) skyve-web META-INF/resources/primefaces-skyve/theme.css imports free saga when the browser
   prefers light and free arya when it prefers dark. Selecting the "dark" colour makes
   UxUis use primefaces-skyve-dark/theme.css, which always imports arya.
2) skyve-war WEB-INF/resources/skyve/editorial/assets/css/themes/<colour>.css
   overrides the --skyve-editorial-* CSS variables (accent, divider, buttons
   and surfaces all derive from these). The view title bar defaults to the
   accent colour; set --skyve-editorial-topbar-bg only when you want different
   chrome. Each palette also supplies dark-safe accent values. dark.css uses those
   accents while overriding the surfaces, borders and text under a dark-mode media query.

The substantial readable Editorial sources live under skyve-web/src/css/editorial and
skyve-web/src/js/editorial. The skyve-web build generates and commits their minified web
resources; generated files must not be edited directly. Small palette, Vue, font and
PrimeFaces alias stylesheets remain readable in their final web-resource locations. The
palettes and Vue integration stay in skyve-war; fonts and the shared PrimeFaces alias stay
in skyve-web.

Put project-specific readable <colour>.css palettes in the WAR so they deploy without
rebuilding skyve-web or running a minifier. No PrimeFaces alias is required per colour.
Full walkthrough: docs/editorial-custom-colour.md.

To install the Ultima theme (>= 7.0.0) on Skyve:
================================================================

1) Drop ultima-theme-7.0.0-jakarta.jar from the Foundry project into WEB-INF/lib/.
2) Drop the ultima-layout/ folder into WEB-INF/resources/ and rename it to layout/.
3) Configure each applicable UxUi with the exact PrimeFaces theme identifier as the third
   argument and the Ultima layout colour as the fourth argument, for example:

	UxUi.newPrimeFaces("phone", "ultima", "ultima-indigo", "indigo")

The third argument is not combined with the fourth. If a different colour is selected, use
the complete theme identifier supplied by the Ultima theme JAR and the matching layout colour.

2) Drop the ecuador-layout/ folder into WEB-INF/resources/ without renaming it.
3) Configure each applicable UxUi with the exact PrimeFaces theme identifier as the third
   argument and the Ecuador layout colour as the fourth argument, for example:

	UxUi.newPrimeFaces("phone", "ecuador", "ecuador-indigo", "indigo")

The third argument is not combined with the fourth. If a different colour is selected, use
the complete theme identifier supplied by the Ecuador theme JAR and the matching layout colour.

To install the Diamond theme on Skyve:
================================================================

1) Drop diamond-theme-7.0.0-jakarta.jar from the Foundry project into WEB-INF/lib/.
2) Drop the diamond-layout/ folder into WEB-INF/resources/ without renaming it.
3) Configure each applicable UxUi with the exact PrimeFaces theme identifier as the third
   argument and the Diamond layout colour as the fourth argument, for example:


	UxUi.newPrimeFaces("phone", "diamond", "diamond-indigo-light", "indigo-light")

The third argument is not combined with the fourth. Use the complete theme identifier supplied
by the Diamond theme JAR and repeat its matching <base-colour>-<scheme> value as the fourth
argument.
